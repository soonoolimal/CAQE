# CLAUDE.md

## Code Style

- Do not align `=` signs across multiple lines. Aligning `#` inline comments is fine.
- All comments and docstrings must be written in English.
- Do not add comments that merely restate what the code does — only comment when the reason (the "why") is non-obvious.
- Write comments as full sentences, not as fragments or dash-separated phrases (e.g., use "Since the dataset is low-noise, ..." not "low-noise dataset — ...").
